the new epoch

发表于 2016-11-20 
edit by liangxj

To improve is to change, to be ferfect is to change often. -Winston Churchill

#1.动机的源头

创建一个自己的blog这个念头完全是机缘巧合，九月末的时候一位大学朋友分享给我了一篇知乎上的文章，讲程序员是怎么找到女朋友的…我…

看完之后顺便翻了翻那个答主其他的回答，有一篇讲旅行的，末尾附上了他的博客，就进去看了看。在这之前我一直对独立博客的印象是一群走火入魔的Geek程序员特殊的社交方式，只谈技术不讲究UI，但是他的博客颠覆了了我对这个名词的理解。他的博客做的相当漂亮而且有不少博文都是旅行和科技产品评测相关的内容，对我这个半吊子计算机学生来说十分友好。因为我又是一个有点热衷视觉效果的人，当时就决定要学着做一个独立博客出来。

#2.搭建博客的过程

九月三十号的时候，因为国庆的长假已经开始，所以那天晚上就熬夜上万网买了一个域名 liangxj.org，然后十月份就出去到南京无锡苏州去玩去了，在途中睡了3个晚上的网吧，然后在玩游戏累了的时候也看了不少基于Hexo+Github搭建博客的教程，然后在7号回合肥之后就着手学这个搭建的过程。

最开始我是在Mac系统中搭建的，但是不知道为啥，估计是电脑太奇葩，总有环境变量有错，连安装启动程序都调试了好久，最后选择放弃在Mac中搭建，转而用Windows系统。不得不说Windows虽然被很多人嫌弃，但是它仍然是我心中最靠谱的系统了，hexo的环境一下子就搭建成功了。

因为十月份有很多考试， 所以在月初的时候弄了下这个之后就搁置了很久，直到上周，也就是11月十五号的样子才开始继续十月份的工作，但是可能是时间隔了太久，有很多配置又发生了变化，后来索性新生成了一个文件，所以看得到博客第一个博文**Hello World” 是十一月十六号生成的。

不得不说，我完全没想到当初简单的一个想法居然实现起来如此的复杂，而且阴差阳错的走到了前端工程师的阵营里面来。无数次的报错无数次的debug，虽然这条路已经是十分成熟，但是从起步到搭建完成的这条路上我不仅绕了远路，而且上每个可能会有错误的地方我全都出过错。每次一坐在电脑前搭建就是一个半天的时间，今天是画了一天的时间全在搭建。

中间也几近无数次的想到放弃，但是一想到没有搭建成功（当时吹过的牛皮）还是迎头就上了(硬着头皮)

#3.搭建博客的目的

就像开头说的那样，最开始的想法只是觉得好看，但是现在花了这么多精力搭建出来了之后，还是想好好的利用一下这个成果。所以我的打算是发一些有意义可以公开的东西在上面，也当是记录生活中的点点滴滴了。

1. 生活中的趣事
2. 可以出来探讨的想法
3. 练习摄影路上的过程

#4.后期的规划

最开始的目的只是追求好看，现在也算是对前端这个领域有了一些了解，感觉还不错，所以我想继续了解然后慢慢改进整个博客的外观和内容，其实现在整个博客的界面对于手机和ipad这些移动设备来说看起来都没有用电脑浏览器起来精致，以后会一边抽空学一边改进外观，不过对于现在的我来说已经够用了，而且我想现在着重在填充内容这项工作上。

然后就是一些附加的第三方功能，多说的评论功能我还是觉得挺重要的，不过尝试过两次都没有成功，后面再尝试一下吧这个重要的功能加进去。

虽然今天感冒咳嗽咳的脑袋都疼，但是今天吃晚饭好开心呀
